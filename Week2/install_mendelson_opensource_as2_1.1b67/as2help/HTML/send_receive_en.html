Connection

It is straight forward to integrate the as2 server to a integration system. We recomment the mendelson business integration. All in- and outbound messages are stored in a directory structure. All following directories are relative to the (defined in the preferences) "messages" directory.

To send messages you have to move them into the outbound directory of your partner. The key data for each partner could be set up in the partner panel.
Receiving the messages is an automatic process.

The following URLs are defined by default:
http://<host>:8080/as2/HttpReceiver
https://<host>:8443/as2/HttpReceiver

Partner directories

Below the message directory the following directories exist:

<Partnername>

<Partnername>/inbox
Stores payloads for every inbound message.
This directory does only exist for local stations.

<Partnername>/outbox/<localstation>
Drop messages you would like to send to a special partner here.
This directory does not exist for local stations.

<Partnername>/error/<localstation>/<date>
If an error occurs by enveloping the message it is moved here.
This directory does not exist for local stations.

<Partnername>/sent/<localstation>/<date>
If the message has been sent successfully the data will be stored here in raw format.
This directory does not exist for local stations.
